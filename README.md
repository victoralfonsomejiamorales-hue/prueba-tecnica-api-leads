## Overview

Professional lead management system with AI analysis powered by Groq. Designed to capture, process, and analyze prospects from multiple sources including Typeform webhooks, with JWT authentication and complete OpenAPI documentation.

### Core Features

- **Lead Management**: Complete CRUD with advanced filtering and pagination
- **AI Analysis**: Executive summaries generated by Groq AI
- **Typeform Integration**: Automatic lead capture via webhooks
- **JWT Authentication**: Robust authentication system
- **Swagger Documentation**: Real-time self-documenting API
- **Docker Ready**: Optimized containers for development and production
- **Rate Limiting**: Abuse protection with configurable throttling

## Quick Start

### Prerequisites

- [Docker](https://docs.docker.com/get-docker/) installed
- [pnpm](https://pnpm.io/installation) installed
- Git

### Setup Instructions

```bash
# 1. Clone the repository
git clone https://github.com/victoralfonsomejiamorales-hue/prueba-tecnica-api-leads.git
cd prueba-tecnica-api-leads

# 2. Install dependencies
pnpm install

# 3. Configure environment variables
cp .env.example .env.development
# Edit .env.development with your values (see Configuration section)

# 4. Start the application
pnpm run docker:dev
```

That's it! The application will be available at:

- **API**: http://localhost:3000/
- **Swagger Documentation**: http://localhost:3000/docs
- **MongoDB**: mongodb://localhost:27017

## Configuration

### Environment Variables

Create a `.env.development` file with the following variables:

| Variable         | Description                 | Example                          |
| ---------------- | --------------------------- | -------------------------------- |
| `NODE_ENV`       | Environment mode            | `development`                    |
| `PORT`           | Application port            | `3000`                           |
| `MONGODB_URI`    | MongoDB connection string   | `mongodb://mongo:27017/leads_db` |
| `GROQ_API_KEY`   | Groq AI API key             | `gsk_xxxxxxxxxxxx`               |
| `GROQ_BASE_URL`  | Groq API base URL           | `https://api.groq.com/openai/v1` |
| `GROQ_MODEL`     | Groq model to use           | `llama3-8b-8192`                 |
| `THROTTLE_TTL`   | Rate limit TTL (ms)         | `60000`                          |
| `THROTTLE_LIMIT` | Rate limit requests per TTL | `10`                             |
| `JWT_SECRET`     | JWT signing secret          | `your-super-secret-key`          |

### Docker Configuration

The project includes optimized Docker configurations:

- **Development**: Hot reload, source mounting, MongoDB exposed
- **Production**: Optimized build, no source mounting, MongoDB secured

## Testing All Endpoints

### 1. Authentication

First, get a JWT token:

```bash
# Login to get token
curl -X POST http://localhost:3000/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "admin@example.com",
    "password": "admin123"
  }'
```

Save the token from the response for subsequent requests.

### 2. Create a Lead

```bash
# Create a new lead (replace TOKEN with your JWT)
curl -X POST http://localhost:3000/leads \
  -H "Content-Type: application/json" \
  -d '{
    "email": "john.doe@example.com",
    "name": "John Doe",
    "role": "user",
    "source": "landing_page",
    "isLead": true,
    "phone": "+1234567890",
    "interestProduct": "Enterprise Software",
    "budget": 5000
  }'
```

### 3. Get All Leads

```bash
# Get paginated leads
curl -X GET "http://localhost:3000/leads?page=1&limit=10" \
  -H "Authorization: Bearer TOKEN"

# Get leads with filters
curl -X GET "http://localhost:3000/leads?source=facebook&page=1&limit=5" \
  -H "Authorization: Bearer TOKEN"
```

### 4. Get Statistics

```bash
# Get lead statistics
curl -X GET http://localhost:3000/leads/stats \
  -H "Authorization: Bearer TOKEN"
```

### 5. Generate AI Summary

```bash
# Generate AI analysis of leads
curl -X POST "http://localhost:3000/leads/ai/summary?page=1&limit=20" \
  -H "Authorization: Bearer TOKEN"
```

### 6. Get Specific Lead

```bash
# Get a lead by ID (replace ID with actual lead ID)
curl -X GET http://localhost:3000/leads/LEAD_ID \
  -H "Authorization: Bearer TOKEN"
```

### 7. Update a Lead

```bash
# Update lead (replace ID and TOKEN)
curl -X PATCH http://localhost:3000/leads/LEAD_ID \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer TOKEN" \
  -d '{
    "name": "John Smith",
    "budget": 7500
  }'
```

### 8. Delete a Lead

```bash
# Delete lead (replace ID and TOKEN)
curl -X DELETE http://localhost:3000/leads/LEAD_ID \
  -H "Authorization: Bearer TOKEN"
```

### 9. Test Typeform Webhook

```bash

curl -X POST http://localhost:3000/leads/webhook \
  -H "Content-Type: application/json" \
  -d '{

      "form_id": "form_123",

        "id": "def_123",

        {
          "type": "email",
```

## Typeform Webhook Integration

### Webhook Features

- **Idempotency**: Duplicate leads are automatically skipped
- **Field Mapping**: Automatic mapping from Typeform answers to lead fields
- **Error Handling**: Proper HTTP status codes and error messages
- **Source Tracking**: All webhook leads are marked with `source: typeform`

### Setting Up Typeform

1. Create a Typeform account and form
2. Configure webhook in Typeform settings
3. Point webhook to: `https://your-domain.com/leads/webhook`
4. Test with the curl command above

## Architecture

### System Flow

```
Typeform Webhook (Public)
    |
    v
Leads Controller (JWT Protected)
    |
    v
Leads Service
    |
    v
Leads Repository
    |
    v
MongoDB
```

### Key Components

- **Controllers**: HTTP request handling with Swagger decorators
- **Services**: Business logic and external integrations
- **Repositories**: Data access layer with Mongoose
- **DTOs**: Request/response validation with class-validator
- **Guards**: JWT authentication and rate limiting
- **Filters**: Global exception handling

## Security Considerations

- **JWT Authentication**: All endpoints except login and webhook are protected
- **Rate Limiting**: Configurable throttling to prevent abuse
- **Input Validation**: Comprehensive DTO validation with class-validator
- **CORS**: Configurable cross-origin resource sharing
- **Helmet**: Security headers middleware
- **Environment Variables**: Sensitive data never committed to version control

## Monitoring & Logging

- **Structured Logging**: NestJS logger with environment context

- **Error Handling**: Global exception filters with proper HTTP status codes
